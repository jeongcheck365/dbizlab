**버전**: 1.0

## 1. 리라이팅 정의

리라이팅 = **의도, 맥락, 글자수를 유지하면서 문장 자체를 새로 작성**

### 올바른 예시
```
원본: "기업의 성장을 위한 최적의 자금 솔루션을 제공합니다"
변경: "비즈니스 확장에 필요한 맞춤형 자금 지원 방안을 안내해 드립니다"
```
→ 의미 동일, 표현 완전히 다름

### 잘못된 예시
```
원본: "KEAI가 기업의 성장을 위한 최적의 자금 솔루션을 제공합니다"
변경: "드림비즈랩이 기업의 성장을 위한 최적의 자금 솔루션을 제공합니다"
```
→ 브랜드명만 바꿈 (SEO 중복 콘텐츠 패널티 위험)

---

## 2. 리라이팅 원칙

| 원칙 | 설명 |
|------|------|
| 의미 동일 | 핵심 메시지 100% 유지 |
| 표현 변경 | 문장구조, 어순, 동의어 전면 활용 |
| 글자수 유지 | ±10% 이내 |
| 자연스러운 한국어 | 번역체 X, 어색함 X |

---

## 3. 작업 영역 구분

### ❌ 건드리지 않는 영역 (그대로 유지)
- **헤더** - 네비게이션, 로고, CTA 버튼
- **입력폼** - 무료심사 신청 폼 전체
- **푸터** - 연락처, 사업자정보, 링크
- **스크립트/JS** - 모든 JavaScript 코드
- **CSS 클래스명** - `keai-*` 등 코드성 클래스

### ✅ 리라이팅 대상 (새로 작성)
- 히어로 섹션 (제목, 설명)
- 서비스 소개 섹션
- 특징/장점 섹션
- 프로세스 설명 섹션
- CTA 섹션 (폼 제외한 텍스트)
- 기타 본문 콘텐츠

## 4. 히어로 이미지 교체

### 기존 (삭제)
