## 4. 이미지 생성 가이드라인

### 4.1 인물 사진 규칙

- **연령**: 30대 한국인
- **직업**: 직장인 또는 CEO
- **복장**: 비즈니스 캐주얼 또는 정장 (자연스러운)
- **표정**: 밝고 신뢰감 있는 미소
- **배경**: 깔끔하고 모던한 한국 사무실
- **조명**: 자연광 또는 소프트 조명
- **스타일**: 스톡 사진 느낌, 과하지 않게

### 4.2 아이콘/일러스트 규칙

- **스타일**: 기존 이미지와 동일한 스타일 유지
- **색상**: 기존 색상 팔레트 분석 후 적용
- **디테일**: 기존 수준과 동일하게

### 4.3 배경 이미지 규칙

- **스타일**: 기존 이미지 분석 후 유사하게
- **분위기**: 전문적, 신뢰감
- **색조**: 기존 이미지 색조 유지

## 5. 프로세스 (TDD 방식)

### Phase 1: 기존 이미지 다운로드 및 분석
```
1. CDN 이미지 16개 다운로드
2. 각 이미지 분석 (스타일, 색상, 구도, 분위기)
3. 분석 결과 문서화
```

### Phase 2: 프롬프트 작성
```
1. 분석 결과 기반 프롬프트 작성
2. 기존 이미지와 유사도 목표: 80% 이상
```

### Phase 3: 이미지 생성 (검증 포함)
```
1. Nano Banana Pro로 이미지 생성
2. [검증] 생성된 이미지 vs 원본 비교
3. [검증] 스타일/분위기 일치 여부 확인
4. 불합격 시 프롬프트 수정 후 재생성
```

### Phase 4: 최종 적용
```
1. 승인된 이미지만 WebP 압축
2. R2 업로드
3. CSS/HTML 적용
```
