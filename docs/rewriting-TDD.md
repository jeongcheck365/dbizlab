## 4. 작업 순서

### Phase 1: 히어로 이미지 생성 (6개)
```
1. Gemini API로 이미지 생성
2. WebP 변환 및 압축
3. public/images/hero/ 저장
4. 각 페이지 히어로 이미지 src 교체
5. 커밋
```

### Phase 2: 페이지별 리라이팅
```
각 페이지마다:
1. 파일 Read (전체 구조 파악)
2. 히어로 섹션 리라이팅
3. 본문 섹션 리라이팅 (순서대로)
4. BAS 규칙 검증
5. 커밋
6. 다음 페이지
```

### Phase 3: 최종 검증
```
1. 전체 BAS 금지 표현 스캔
2. 브랜드명 일관성 확인
3. 프로덕션 배포
4. 라이브 확인
```
